Race Condition Analysis:

Thread A (Background ReceiveLoop):
  HandleServerMessage() {
    AvailableModels = models;  // Line 681: Reassigns List<string> reference
  }

Thread B (UI/Main):
  get AvailableModels {
    return _bridgeClient.AvailableModels;  // Line 265: Returns current reference
  }
  
Thread C (UI/Main accessing Count):
  _bridgeClient.AvailableModels.Count  // Line 264: Reads Count property

Scenario 1: Tearing
  - Thread C reads AvailableModels reference
  - Thread A reassigns AvailableModels to new list
  - Thread C calls .Count on old list
  Result: No issue - reading old list is fine

Scenario 2: Collection modification during iteration
  - Thread B/C gets reference to list
  - Thread A replaces entire reference (doesn't modify existing list)
  - Thread B/C continues with old list reference
  Result: No issue - no modification of existing list

VERDICT: Safe because:
1. List<T> reference assignment is atomic on .NET
2. Old list is never modified (replaced, not mutated)
3. Worst case: UI briefly shows stale model list until next OnStateChanged
